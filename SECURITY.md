# Security policy

## Segnalare una vulnerabilità

Usa una [private security advisory](https://github.com/walterzannoni90-netizen/piattaformaipersonale/security/advisories/new). Includi versione, impatto, prerequisiti e una riproduzione minima. Non inserire token, dati personali o credenziali reali.

Non aprire una issue pubblica per vulnerabilità sfruttabili. Il maintainer deve confermare la ricezione, valutare severità e coordinare correzione e divulgazione prima di pubblicare dettagli.

## Ambito

Sono in ambito autenticazione, isolamento tra account, segreti, upload/download, worker Python, fetch web, webhook e API applicative. Problemi esclusivi di servizi terzi vanno segnalati anche al rispettivo fornitore.

## Versioni supportate

Fino al primo rilascio stabile è supportato soltanto l’ultimo commit pubblicato sul branch di produzione.
